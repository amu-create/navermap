네이버 클라우드 플랫폼 Maps API 설정 가이드

인증 오류 문제: 네이버 지도 API는 보안상의 이유로 등록된 웹 서비스 URL에서만 API를 사용할 수 있습니다. 현재 도메인이 네이버 클라우드 플랫폼에 등록되어 있지 않아 인증 오류가 발생했습니다.

1. 네이버 클라우드 플랫폼 로그인 및 애플리케이션 접근

  1. 네이버 클라우드 플랫폼에 로그인합니다.
  2. 로그인 후 우측 상단의 콘솔 버튼을 클릭합니다.
  3. 왼쪽 메뉴에서 AI・Application > Maps를 선택합니다.
  4. Maps 서비스 페이지에서 Application 탭을 클릭합니다.

2. Web 서비스 URL 등록

  1. 등록된 애플리케이션을 찾아 클릭합니다. (Client ID: me6liq8hnh)
  2. Web 서비스 URL 섹션을 찾습니다.
  3. 추가 버튼을 클릭하고 다음 URL을 각각 등록합니다:
    • GitHub Pages 환경: https://amu-create.github.io
    • 로컬 테스트 환경: http://localhost:8000
  4. 저장 버튼을 클릭합니다.

중요: 도메인을 등록할 때 다음 사항에 주의하세요:

  • URL 끝에 슬래시(/)를 포함하지 마세요. 예: https://amu-create.github.io (O), https://amu-create.github.io/ (X)
  • 변경사항이 적용되는 데 최대 10-15분이 소요될 수 있습니다.
  • 로컬 테스트용으로는 http://localhost 또는 http://localhost:포트번호 형태로 등록합니다.

3. API 인증 확인

  1. 도메인 등록 후 약 5-15분 정도 기다립니다.
  2. 웹 브라우저를 새로고침하거나 다시 접속해봅니다.
  3. 로컬 환경에서 테스트하려면:
    python -m http.server
    를 실행하고 http://localhost:8000으로 접속합니다.
  4. GitHub Pages 환경에서 테스트하려면 https://amu-create.github.io/navermap/로 접속합니다.

4. 문제 해결

인증 오류가 계속 발생하는 경우

문제 해결 방법
등록한 URL과 접속 URL이 정확히 일치하지 않음
  • 프로토콜(http/https)이 정확히 일치하는지 확인
  • 포트번호가 있는 경우 포트번호까지 정확히 등록
  • 대소문자 구분에 주의
변경사항 적용 시간 부족 네이버 클라우드 플랫폼에서 설정 변경 후 최대 15분까지 기다려보세요.
브라우저 캐시 문제 브라우저 캐시를 비우거나 시크릿/프라이빗 모드에서 접속해보세요.
클라이언트 ID 문제 올바른 클라이언트 ID를 사용하고 있는지 확인하세요.

로컬 서버 실행 중인지 확인: 로컬에서 테스트할 때는 반드시 웹 서버가 실행 중이어야 합니다. 터미널에서 python -m http.server 명령어가 실행 중인지 확인하세요.
